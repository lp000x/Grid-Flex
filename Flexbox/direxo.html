Flex Direction: Container

Existem 4 configurações que podem ser aplicadas como flex direction:

  • Flex-direction: row - faz os itens dentro do container serem dispostos em linha, por padrão, da esquerda para a direita.
  • Flex-direction: column - faz os itens dentro do container serem dispostos em coluna por padrão, de cima para baixo.

Eu repeti e coloquei destacado "por padrão". Isso porque existem uma variação para coluna e d linha:

  • Flex-direction: row-reverse - ainda mantém a disposição dos itens em linha, mas em vez da esquerda para a direita, os itens vão ser exibidos da direita para esquerda.
  • Flex-direction: column-reverse - ainda mantém a dispoição dos itens em coluna, mas de baixo para cima em vez de cima para baixo.

AXIS (EIXOS)

Quando eu crio um display flex com um flex-direction, automaticamente são criados eixos.

Esses eixos dependem se é uma coluna ou uma linha e, além disso, se é reverse ou sem reverse.

Quais São Os Eixos?

Os eixos são: main e transversal.

main-axis: o eixo main é criado horizontalmente cruzando todos o container. É como se fosse criada uma linha invisivel posta horizontalmente bem na metade do container. transversal-axis: também chamado de cross-axis, esse eixo é criado de cima para baixo do container. Assim como o main, é uma linha invisivel.

Porém, quando eu coloco row-reverse ou column-reverse, não é apenas a disposição dos elementos que mudam, mas a orientação dos eixos também. Assim como, ao definir row-reverse, os elementos são dispostos da direita para a esquerda, o eixo também muda, indo da direita para a esquerda. Via de regra, os eixos acompanham a direção da dispoção dos elementos, dispoição essa definida pelo reverse.

Para que saber localizar os eixos são importantes? Porque o justify conten se baseia nos eixos.

Main start, Main End, Cross Start, Cross End

Se o eixo diz respeito à direção que os eixos apontam, o start e end (de main e cross) são os exatos pontos onde o eixo começa e termina. Por padrão, esses pontos estão no container.
