Container X Elementos

Toda vez que eu criar um container e colocar nele elementos, que vai receber o display flex é o container. É possível tornar os elementos dentro da caixa display flex, mas isso não será visto nesse curso.

Order

Order é uma propriedade que dispõe os elementos em uma determinada ordem. Cada elemento vai receber um valor e eles serão organizados com base nele. Por padrão, o valor dos elementos é 0, ou seja, não tem preferencia de ordenamento.

45
934
1
74
89
32
7

Neste exemplo, mesmo estando em ordem diferente no HTML os números seguiram a ordem na tela.

Na propriedade order, eu adiciono qualquer número. Os números negativos vem primeiro, seguido do 0 (lembrando que todo ou todos os itens sem valor declarado vem por padrão com valor 0) e os maiores vem depois.

Mas é no seguinte caso: quem vai aparecer primeiro, o B ou E? Os dois estão com valor 0.

A
B
C
D
E
F
G

Via de regra, na ordem vai aparecer primeiro os negativos, depois o primeiro 0 que vem no HTML, depois o segundo 0 e assim com todos eles. O "ordenamento" dos zeros é feito de acordo com qual item vem primeiro no HTML. Nesse caso, o B veio primeiro, então ele aparecerá antes do E.

aling-self

O aling-self é aplicado no eixo transversal, ou seja, no eixo vertical.

aling-self tem 5 valores:

AUTO
START
END
CENTER
STRECH
  • Auto: vai herdar a caracteristica de alinhamento de seu pai
  • flex-start: é o alinhamento no ponto que demarca o começo do eixo transversal
  • flex-end: é o alinhamento no ponto que demarca o final do eixo transversal
  • center: vai alinhar o item no centro do container
  • strech: vai fazer com que o item ocupe toda a altura do container

Eu posso colocar align-items no container como uma configuração geral, mas caso eu altere diretamente no item ela vai se sobrepor.
