Shorthand flex

Flex-grow, flex-shrink e flex-basis podem ser declarados com apenas 1 propriedade: flex. Assim como outros shorthands, existe uma ordem na declaração:

  • Grow
  • Shrink
  • Basis

Padrões de Uso

Embora cada caso é um caso, acaba se tornando muito comum e recorrente o uso dos mesmos valores de grow, shrink e basis. Por isso, existem certos valores padrões:

Flex: 0 1 auto (grow 0, shrink 1 e basis auto)

Esse conjunto nem precisa ser declarado, pois ele é os valores padrões de cada uma dessas propridades.Lembrando: grow sempre tem valor 0, shrink sempre tem valor 1 e flex-basis sempre tem valor auto.

Outra forma de declarar o valor padrão (e isso para qualquer propriedade, não só flex) é usar o initial (flex: initial). Initial faz com que a propriedade seja setada em seus valores padrões.

flex: 0 0 auto (grow: 0, shrink: 0, basis: auto)

Esse conjunto torna o item inflexivel ao container, se adaptando somente ao conteúdo inserido dentro dele.

Outra forma de declarar essa "caracteristica" de inflexibilidade é usar o flex: none.

Flex: 1 1 auto (grow: 1, shrink: 1, basis: auto)

Esse é o oposto do anterior: o item vai se tornar totalmente flexivel. Ele vai crescer se o container crescer (grow 1), vai encolher se o container encolher (shrink 1) e o item vai se adaptar no tamanho conforme o conteúdo dentro dele (flex-basis: auto).

Uma outra forma de declarar esses valores é usar flex: auto.

Declarar apenas 1 valor

Se eu declarar apenas: flex: 3, o que acontece?

Nesse caso, será interpretado que grow terá o valor de 3, e todos os outros (shrink e basis) terão seus valores padrões. O mesmo vale para qualquer outro valor e só vale para grow.
