Alinhamento Nos Eixos

Este tema ainda é aplicável ao container, ao elemento pai. Ou seja, a caixa que "abriga" elementos.

Como dito antes, existem 2 eixos: o transversal (cross), que é de cima para baixo (ou de baixo para cima se for reverse) e principal (main), que é horizontal da esquerda para direita (ou direita para esquerda se for reverse).

Main = justify-content

Caso eu queira mexer no alinhamento horizontal, eu utilizo a propriedade justify-content. Esta é a função do justify-content: alinhar o item horizontalmente.

Porém é sempre preciso se atentar se estou utilizando um flex-direction row ou column, pois isso afeta a aplicação dessa propriedade.

Main Start e Main end

Quando digo apenas "eixo principal" (ou main axis), eu posso querer me referir apenas à orientação desse eixo, ou seja, se é horizontal ou vertical, e sendo horizontal, se vai da esquerda para direita ou o contrário.

Porém, para o justify-content é preciso exatidão, uma exatidão que só a direção não oferece. Para isso, se utiliza outro conceito já mostrado: main start e main end.

O main start marca onde a linha horizontal começa, enquanto que o main end marca onde termina. Essas 2 estão no começo e final da largura do container.

justify-content: flex-start

O valor flex-start define que o elemento será posicionado no ponto onde começa o eixo. Sempre na esquerda? Não, como já dito, para onde a direção estiver apontando

justify-content: flex-start (row normal)

justify-content: flex-start (row-reverse)

justify-content: flex-end

O valor flex-end é o oposto: alinha o conteúdo no main end, ou seja, no ponto onde o eixo termina (lembrando onde o eixo começa e termina depende totalmente se estou usando normal ou reverse)

justify-content: flex-end (row normal)

justify-content: flex-end (row-reverse)

justify-content: center

Outro valor é o center, que posiciona o elemento no centro do container:

justify-content: space-between

space-between coloca um elemento no main-start, outro elemento no main-end e o outro (ou os outros) no meio, com um espaçamento uniforme (o mesmo espaço) entre eles.

justify-content: space-evenly

space-evenly faz com que todos os elementos tenha o mesmo espaçamento entre si. Ele faz isso logo no main start, o espaço entre entre o primeiro item o main start será o mesmo para todos (com base em quantos itens tem e o tamanho do container)

justify-content: space-around

space-around faz uma conta diferente. Ele pega o container, divide o espaço baseado na quantidade de elementos (ou seja, se tem 3 elementos, divide o espaço do container por 3, se tem 4 elementos, divide o espaço por 4, e a mesma lógica para qualquer valor), e centraliza o elemento no meio desses espaços.

O Que Muda Entre Os justify-content?

Pode parecer que é apenas onde os elementos serão posicionados, mas não é bem por ai. Na verdade, cada jusitify-content calcula o espaço do container de forma diferente. Essa diferença nos cálculos é o que muda o posicionamento dos elementos dentro do container.

Transversal, Cross Start e Cross End

Outro eixo visto foi o transversal (ou cross). Esse eixo é vertical e vai de cima para baixo se a direção estiver normal (ou com wrap-reverse) e de baixo para cima se estiver em reverse.

Assim como o main axis, ele tem seus pontos onde demarca seu começo e seu fim, e esses pontos não são cima = começo e baixo = fim. Como dito várias vezes, o começo e o final depende se a direção está normal ou reverse.

E DETALHE: Tanto a direção row quanto a direção column tem um eixo main e cross. Justify content altera na horizontal, enquanto que aling-items na vertical.

Diferente do main axis, o cross axis usa aling-items para definir o posicionamento dos itens.

Para alterar o posicionamento aling-items, é preciso ter um container com uma altura um pouco maior.

Strech

Por padrão, o navegador define um container com valor strech. Isso significa que se o container tiver uma altura maior que a do elemento, o elemento vai se aumentar seu tamanho também.

aling-items: flex-start

Faz o elemento se posicionar no começo do eixo vertical

align-items: flex-start (row normal)

align-items: flex-start (row reverse)

align-items: flex-end

align-items: center

aling-items E jusitify-content

Olhando os 2 exemplos, é possível perceber uma coisa:

  • Todos os jusitify-content permaneceram só na parte superior do container
  • Todos os align-items permaneceram somente na parte esquerda do container

Isso acontece por algo já explicado: aling-items altera somente o eixo vertical, enquanto que jusitify-content somente o eixo horizontal. Caso eu queira alterar as duas coisas, é preciso usar as 2 propriedades.
